# Генератор изображений через RunWare — Инструкция по созданию

## Что это и зачем

Веб-приложение: пользователь вводит текстовый промпт, нажимает кнопку — и получает сгенерированное изображение. Приложение отправляет запрос в RunWare, RunWare передаёт его в OpenAI GPT Image 2, изображение создаётся на серверах OpenAI и возвращается обратно.

RunWare — посредник (API-роутер для изображений). GPT Image 2 — закрытая модель, обратиться к ней напрямую сложно. RunWare решает эту проблему. Пример из конспекта: приложение MyBanana генерирует картинки именно по такой схеме.

## Стек

| Компонент | Технология |
|-----------|-----------|
| Backend | Python + Flask |
| Генерация изображений | RunWare API → GPT Image 2 |
| Frontend | HTML + Jinja2 |
| HTTP-запросы | requests |
| Конфиг | python-dotenv |

## Важные нюансы

- RunWare работает по кредитной системе — каждый запрос стоит денег, начинать с маленького баланса
- Если изображение не загружается в `<img>` — проверить CORS-заголовки или сохранять файл локально
- Timeout Flask по умолчанию может не дождаться ответа RunWare — увеличить до 60 секунд: `requests.post(..., timeout=60)`
- Промпт на английском языке обычно даёт лучший результат для генерации изображений
